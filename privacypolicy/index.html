Privacy Policy for WFT Helper Chrome Web Extension

**Effective Date:** October 2, 2025 (Updated)

This Privacy Policy describes how the **WFT Helper Chrome Web Extension** ("the Extension") handles user data. The Extension's sole function is to synchronize your work schedule from The Home Depot's proprietary Workforce Tools portal to your personal Google Calendar.

The core principle of this Extension's design is local data processing and user privacy. We do not transmit user data to external servers outside of the Google Calendar API call initiated by the user.

1. Data Collection and Processing

The WFT Helper Extension operates primarily within your local web browser environment. **We do not collect, transmit, store, or sell any user data on our own external servers.** All processing and storage of sensitive information are performed locally on your device, with the exception of required interactions with the Google Calendar API.

A. Data Processed Locally (Non-Google User Data)

The Extension processes the following data types directly within your browser:

Data Type Source Purpose of Processing
**Work Schedule Details** (Date, Time, Location) The Home Depot Workforce Tools portal (`https://wft.homedepot.com/`) via `webRequest` interception. Used solely to format shifts into calendar events and send them to your Google Calendar.
**Workforce User Data** (Name) The Home Depot Workforce Tools API endpoints. Used solely to personalize the name of the dedicated Google Calendar (e.g., "John Doe's WFT Calendar").

B. Google User Data Accessed and Stored Locally

The Extension uses the `identity` permission and `chrome.storage` to handle the following Google user data, which is only processed for the function of syncing your schedule:

  • **Google Access Token (Google User Data):** A temporary token used to authorize access to the Google Calendar API on your behalf. This is stored locally in `chrome.storage` and automatically expires.
  • **Google Calendar ID (Google User Data):** The unique identifier for the dedicated WFT Helper Calendar created in your Google account. Stored locally to ensure future events are synced to the correct calendar.
  • **Google User Email (Google User Data):** Stored locally for validation purposes, ensuring the user is syncing to the calendar created with the correct Google account.
  • **Work Shift Events:** The processed schedule data (date, time, location) is created as an event and **shared with the Google Calendar API** to be written to your personal Google Calendar. This is the only instance where processed data leaves your local device and is sent to a third-party (Google).

2. Limited Use of Google User Data

The WFT Helper Extension's use of information received from Google APIs will adhere to the **Google API Services User Data Policy, including the Limited Use requirements.**

  • **No Sale of Data:** We will **not** sell Google user data (including Access Tokens, Calendar IDs, or Email Addresses) to third parties.
  • **Limited Transfer:** We will **not** transfer Google user data to others, except as necessary to provide or improve the core, user-facing functionality of the Extension (i.e., syncing your work schedule to Google Calendar), comply with applicable law, or as part of a merger/acquisition.
  • **No Advertising or Profiling:** We will **not** use or transfer Google user data to serve users personalized, retargeted, or interest-based advertising.
  • **No AI Training:** We will **not** use Google user data for training, modeling, or improving generative AI/machine learning models.

3. Data Security and Protection

We employ appropriate technical and organizational measures designed to protect the confidentiality and integrity of your data:

  • **OAuth 2.0:** We use Google's secure and standard OAuth 2.0 protocol for authentication. This means we never see or store your Google password.
  • **Secure Local Storage:** All sensitive Google user data (Access Token, Calendar ID, Email) is stored in the browser's `chrome.storage` API, which is isolated to the Extension and protected by the browser's security model.
  • **Encrypted Transmission:** All communications with the Google Calendar API are conducted over secure, industry-standard encrypted connections (HTTPS/SSL).

4. Data Retention and Deletion

We only retain data for as long as necessary to fulfill the services requested by the user.

  • **Authentication Tokens:** Google Access Tokens are temporary and either automatically expire (typically within an hour) or are securely refreshed as needed to maintain service functionality. The Refresh Token is stored locally until the Extension is uninstalled.
  • **Local Data Deletion:** Upon uninstallation of the WFT Helper Extension from your Chrome browser, all locally stored data, including the Google User Email, Calendar ID, and all authentication tokens, is immediately and permanently deleted from your device's `chrome.storage`.
  • **Calendar Events:** The calendar events created in your Google Calendar are under your direct control and retention policy. Deleting the Extension will **not** automatically delete the events it created. You must delete the events or the dedicated calendar yourself via Google Calendar settings.

5. Permissions Justification

The Extension requests the following permissions to fulfill its sole function of synchronizing your schedule:

  • **Host Permission (`https://wft.homedepot.com/`):** Required to inject the content script and use `webRequest` to capture internal schedule data endpoints.
  • **`webRequest`:** Required for monitoring network traffic **exclusively on the Home Depot domain** to securely obtain the raw schedule data.
  • **`identity`:** Required to initiate the secure Google OAuth flow, allowing the user to grant access to the Google Calendar API.
  • **`storage`:** Required for local, secure storage of configuration data and authentication tokens.
  • **`activeTab`:** Required to ensure the synchronization process is initiated successfully upon user interaction (clicking the extension icon).

6. Children's Privacy

The WFT Helper Extension is not intended for use by individuals under the age of 18. We do not knowingly process or store data from children.

7. Changes to This Policy

We may update our Privacy Policy from time to time. We will notify users of any material changes by updating the effective date of this policy and providing prominent notice within the extension or extension store listing. Continuing to use the Extension after changes are made signifies your acceptance of the revised policy.
